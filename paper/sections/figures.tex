    \textbf{\quantmsdiann{}: SDRF-driven workflow, architecture, and runtime scaling.}
    (\textbf{a})~Conceptual overview: diverse public DIA datasets (bulk,
    single-cell plexDIA, spatial, phosphoproteomics), each described by one
    standardised SDRF, are reanalysed by a single nf-core/\texttt{DIA-NN}
    workflow parallelised across HPC/cloud and exported to one harmonised,
    queryable \qpx{} archive.
    (\textbf{b})~Subway diagram of the \quantmsdiann{} workflow: mandatory
    modules as filled circles, optional/skippable modules as open circles.
    Per-file parallel stages (red; file preparation, preliminary
    calibration, individual search) run as one task per raw file, whereas
    collective stages (green; in-silico library generation, empirical
    library assembly, final quantification, MSstats conversion, and
    \texttt{pmultiqc} reporting) run once over the full cohort.
    (\textbf{c})~Workflow wall-clock time versus the number of cluster nodes
    (Nextflow \texttt{executor.queueSize}) on the PXD071075 single-cell
    cohort.
    (\textbf{d})~Wall-clock time to finish each reanalysis, one bar per
    dataset, ordered by cohort size (number of MS data files) and coloured
    by instrument family. For proteome-scale cohorts, time-to-finish stays
    within a few hours regardless of file count because per-file work is
    parallelised; the longest run is a phosphoproteomics cohort (PXD049692),
    where generating the deep variable-modification spectral library
    dominates the wall-clock.
  }
  \label{fig:architecture}
\end{figure}
